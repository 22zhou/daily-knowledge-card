# Android相册按日期分组功能实现指南

> 本章节基于用户项目结构（gallery模块），详细讲解如何实现相册照片按日期分类展示功能。

## 4.1 功能需求分析

### 4.1.1 用户场景
- 用户打开相册，期望按"今天"、"昨天"、"上周"、"更早"等时间维度分组
- 每个分组下显示该日期的所有照片
- 分组标题显示日期和照片数量

### 4.1.2 技术方案选择

**方案对比**：

| 方案 | 优点 | 缺点 |
|-----|------|------|
| MediaStore直接查询 | 实现简单 | 难以按日期分组 |
| Room数据库缓存 | 支持复杂查询 | 需要维护数据同步 |
| 本地缓存+分组 | 性能最优 | 实现复杂度中等 |

**推荐方案**：结合Room缓存与MediaStore原始数据，实现高效分组展示。

```kotlin
/**
 * 媒体分组实体类

 */
data class MediaGroup(
    val id: String,                 // 分组唯一ID（如"2026-04-19"）
    val title: String,               // 分组标题（如"今天"、"4月19日"）
    val date: Long,                  // 分组日期时间戳

) {

     * 生成格式化标题
     * 根据日期与今天的距离，返回友好格式
     */
    fun getDisplayTitle(): String {
        val now = System.currentTimeMillis()
        val diffDays = ((now - date) / (1000 * 60 * 60 * 24)).toInt()
        
        return when (diffDays) {
            0 -> "今天"
            1 -> "昨天"
            in 2..7 -> "${diffDays}天前"
            in 8..30 -> "${diffDays / 7}周前"
            else -> {
                val calendar = java.util.Calendar.getInstance()
                calendar.timeInMillis = date
                "${calendar.get(java.util.Calendar.MONTH) + 1}月${calendar.get(java.util.Calendar.DAY_OF_MONTH)}日"
            }
        }
    }
}
```

## 4.3 MediaStore数据源设计

### 4.3.1 MediaStore查询工具类

```kotlin
// gallery/data/remote/MediaStoreHelper.kt
package com.example.gallery.data.remote

import android.content.ContentResolver
import android.content.ContentUris
import android.content.Context
import android.database.Cursor
import android.net.Uri
import android.os.Build
import android.provider.MediaStore
import com.example.gallery.data.local.model.MediaItem
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.withContext

/**
 * MediaStore数据源辅助类
 * 负责从系统媒体库读取原始数据
 */
class MediaStoreHelper(private val context: Context) {
    
    private val contentResolver: ContentResolver = context.contentResolver
    
    // 外部存储的媒体URI
    private val externalUri: Uri = if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.Q) {
        MediaStore.Images.Media.getContentUri(MediaStore.VOLUME_EXTERNAL)
    } else {
        MediaStore.Images.Media.EXTERNAL_CONTENT_URI
    }
    
    /**
     * 查询所有图片和视频
     * 使用协程避免阻塞主线程
     */
    suspend fun queryAllMedia(): List<MediaItem> = withContext(Dispatchers.IO) {
        val mediaList = mutableListOf<MediaItem>()
        
        // 投影：指定要查询的列
        val projection = arrayOf(
            MediaStore.Images.Media._ID,
            MediaStore.Images.Media.DATA,
            MediaStore.Images.Media.DISPLAY_NAME,
            MediaStore.Images.Media.MIME_TYPE,
            MediaStore.Images.Media.SIZE,
            MediaStore.Images.Media.WIDTH,
            MediaStore.Images.Media.HEIGHT,
            MediaStore.Images.Media.DATE_TAKEN,
            MediaStore.Images.Media.DATE_ADDED,
            MediaStore.Images.Media.DATE_MODIFIED,
            MediaStore.Images.Media.BUCKET_ID,
            MediaStore.Images.Media.BUCKET_DISPLAY_NAME,
            MediaStore.Images.Media.ORIENTATION
        )
        
        // 查询条件：只查询图片和视频
        val selection = "${MediaStore.Images.Media.MIME_TYPE} LIKE 'image/%' " +
                        "OR ${MediaStore.Images.Media.MIME_TYPE} LIKE 'video/%'"
        
        // 按拍摄日期降序排列（最新的在前）
        val sortOrder = "${MediaStore.Images.Media.DATE_TAKEN} DESC"
        
        var cursor: Cursor? = null
        try {
            cursor = contentResolver.query(
                externalUri,
                projection,
                selection,
                null,
                sortOrder
            )
            
            cursor?.let {
                val idColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media._ID)
                val pathColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.DATA)
                val nameColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.DISPLAY_NAME)
                val mimeColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.MIME_TYPE)
                val sizeColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.SIZE)
                val widthColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.WIDTH)
                val heightColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.HEIGHT)
                val dateTakenColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.DATE_TAKEN)
                val dateAddedColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.DATE_ADDED)
                val dateModifiedColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.DATE_MODIFIED)
                val bucketIdColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.BUCKET_ID)
                val bucketNameColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.BUCKET_DISPLAY_NAME)
                val orientationColumn = it.getColumnIndexOrThrow(MediaStore.Images.Media.ORIENTATION)
                
                while (it.moveToNext()) {
                    val mediaItem = MediaItem(
                        id = it.getLong(idColumn),
                        path = it.getString(pathColumn) ?: continue,
                        name = it.getString(nameColumn) ?: "unknown",
                        mimeType = it.getString(mimeColumn) ?: "image/*",
                        size = it.getLong(sizeColumn),
                        width = it.getInt(widthColumn),
                        height = it.getInt(heightColumn),
                        // DATE_TAKEN可能为0，使用DATE_ADDED作为备选
                        dateTaken = it.getLong(dateTakenColumn).takeIf { d -> d > 0 } 
                            ?: (it.getLong(dateAddedColumn) * 1000),
                        dateAdded = it.getLong(dateAddedColumn) * 1000,
                        dateModified = it.getLong(dateModifiedColumn) * 1000,
                        bucketId = it.getString(bucketIdColumn) ?: "default",
                        bucketName = it.getString(bucketNameColumn) ?: "默认相册",
                        orientation = it.getInt(orientationColumn)
                    )
                    mediaList.add(mediaItem)
                }
            }
        } finally {
            cursor?.close()
        }
        
        mediaList
    }
    
    /**
     * 获取指定媒体文件的缩略图URI
     */
    fun getThumbnailUri(mediaId: Long): Uri {
        return ContentUris.withAppendedId(externalUri, mediaId)
    }
}
```

## 4.4 分组逻辑实现

### 4.4.1 分组管理器

```kotlin
// gallery/domain/usecase/MediaGroupUseCase.kt
package com.example.gallery.domain.usecase

import com.example.gallery.data.local.model.MediaGroup
import com.example.gallery.data.local.model.MediaItem
import java.text.SimpleDateFormat
import java.util.*

/**
 * 媒体分组用例
 * 负责将媒体列表按日期分组
 */
class MediaGroupUseCase {
    
    // 日期格式化：用于生成日期分组ID
    private val dateFormat = SimpleDateFormat("yyyy-MM-dd", Locale.getDefault())
    
    // 日历对象（复用避免频繁创建）
    private val calendar = Calendar.getInstance()
    
    /**
     * 按日期分组媒体列表
     * @param mediaList 媒体列表（应按dateTaken降序排列）
     * @return 分组列表
     */
    fun groupByDate(mediaList: List<MediaItem>): List<MediaGroup> {
        if (mediaList.isEmpty()) return emptyList()
        
        // 按日期分组
        val groupedByDate = mediaList.groupBy { item ->
            // 使用dateTaken获取日期字符串作为分组键
            calendar.timeInMillis = item.dateTaken
            calendar.set(Calendar.HOUR_OF_DAY, 0)
            calendar.set(Calendar.MINUTE, 0)
            calendar.set(Calendar.SECOND, 0)
            calendar.set(Calendar.MILLISECOND, 0)
            dateFormat.format(Date(calendar.timeInMillis))
        }
        
        // 转换为MediaGroup列表
        return groupedByDate.map { (dateKey, items) ->
            val firstItem = items.first()
            calendar.timeInMillis = firstItem.dateTaken
            
            MediaGroup(
                id = dateKey,
                title = generateTitle(dateKey),
                date = firstItem.dateTaken,
                count = items.size,
                coverPath = firstItem.path,
                items = items
            )
        }.sortedByDescending { it.date }  // 按日期降序排列
    }
    
    /**
     * 根据日期键生成友好标题
     */
    private fun generateTitle(dateKey: String): String {
        return try {
            val date = dateFormat.parse(dateKey)
            date ?: return dateKey
            
            calendar.timeInMillis = date.time
            val targetDay = calendar.get(Calendar.DAY_OF_YEAR)
            val targetYear = calendar.get(Calendar.YEAR)
            
            val today = Calendar.getInstance()
            val todayDay = today.get(Calendar.DAY_OF_YEAR)
            val todayYear = today.get(Calendar.YEAR)
            val diffDays = (todayDay - targetDay) + (todayYear - targetYear) * 365
            
            when {
                diffDays == 0 -> "今天"
                diffDays == 1 -> "昨天"
                diffDays in 2..7 -> "${diffDays}天前"
                diffDays in 8..30 -> "${diffDays / 7}周前"
                diffDays in 31..365 -> "${diffDays / 30}个月前"
                else -> dateKey
            }
        } catch (e: Exception) {
            dateKey
        }
    }
}
```

```kotlin
// gallery/ui/home/HomeViewModel.kt
package com.example.gallery.ui.home

import android.app.Application
import androidx.lifecycle.AndroidViewModel
import androidx.lifecycle.viewModelScope
import com.example.gallery.data.local.db.GalleryDatabase
import com.example.gallery.data.local.model.MediaGroup
import com.example.gallery.data.remote.MediaStoreHelper
import com.example.gallery.domain.usecase.MediaGroupUseCase
import kotlinx.coroutines.flow.*
import kotlinx.coroutines.launch

class HomeViewModel(application: Application) : AndroidViewModel(application) {
    
    // 依赖注入（简化版，实际推荐使用Hilt/Koin）
    private val mediaStoreHelper = MediaStoreHelper(application)
    private val database = GalleryDatabase.getInstance(application)
    private val mediaDao = database.mediaDao()
    private val groupUseCase = MediaGroupUseCase()
    
    // UI状态
    private val _uiState = MutableStateFlow<HomeUiState>(HomeUiState.Loading)
    val uiState: StateFlow<HomeUiState> = _uiState.asStateFlow()
    
    // 媒体分组列表
    val mediaGroups: StateFlow<List<MediaGroup>> = mediaDao.getAllMediaByDate()
        .map { items -> groupUseCase.groupByDate(items) }
        .stateIn(
            scope = viewModelScope,
            started = SharingStarted.WhileSubscribed(5000),
            initialValue = emptyList()
        )
    
    // 最后同步时间（用于增量更新）
    private var lastSyncTime: Long = 0L
    
    init {
        // 启动时同步媒体数据
        syncMediaData()
    }
    
    /**
     * 同步媒体数据
     * 从MediaStore读取并缓存到Room
     */
    fun syncMediaData() {
        viewModelScope.launch {
            _uiState.value = HomeUiState.Loading
            
            try {
                // 从MediaStore读取所有媒体
                val mediaList = mediaStoreHelper.queryAllMedia()
                
                // 增量更新：只更新变化的媒体
                if (lastSyncTime > 0) {
                    val modifiedMedia = mediaList.filter { it.dateModified > lastSyncTime }
                    if (modifiedMedia.isNotEmpty()) {
                        mediaDao.insertAll(modifiedMedia)
                    }
                } else {
                    // 首次全量同步
                    mediaDao.deleteAll()
                    mediaDao.insertAll(mediaList)
                }
                
                lastSyncTime = System.currentTimeMillis()
                _uiState.value = HomeUiState.Success
                
            } catch (e: Exception) {
                _uiState.value = HomeUiState.Error(e.message ?: "同步失败")
            }
        }
    }
    
    /**
     * 刷新数据（手动触发）
     */
    fun refresh() {
        lastSyncTime = 0L  // 强制全量同步
        syncMediaData()
    }
}

/**
 * UI状态密封类
 */
sealed class HomeUiState {
    data object Loading : HomeUiState()
    data object Success : HomeUiState()
    data class Error(val message: String) : HomeUiState()
}
```

```kotlin
// gallery/ui/detail/adapter/MediaDetailAdapter.kt
package com.example.gallery.ui.detail.adapter

import android.view.LayoutInflater
import android.view.ViewGroup
import androidx.recyclerview.widget.DiffUtil
import androidx.recyclerview.widget.ListAdapter
import androidx.recyclerview.widget.RecyclerView
import com.bumptech.glide.Glide
import com.example.gallery.data.local.model.MediaItem
import com.example.gallery.databinding.ItemMediaGridBinding

/**
 * 媒体网格适配器
 * 用于展示单个分组内的所有媒体
 */
class MediaDetailAdapter(
    private val onItemClick: (MediaItem, Int) -> Unit,
    private val onItemLongClick: (MediaItem) -> Boolean  // 返回true表示消费事件
) : ListAdapter<MediaItem, MediaDetailAdapter.MediaViewHolder>(MediaDiffCallback()) {
    
    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): MediaViewHolder {
        val binding = ItemMediaGridBinding.inflate(
            LayoutInflater.from(parent.context),
            parent,
            false
        )
        return MediaViewHolder(binding)
    }
    
    override fun onBindViewHolder(holder: MediaViewHolder, position: Int) {
        holder.bind(getItem(position))
    }
    
    inner class MediaViewHolder(
        private val binding: ItemMediaGridBinding
    ) : RecyclerView.ViewHolder(binding.root) {
        
        fun bind(item: MediaItem) {
            // 加载缩略图
            Glide.with(binding.ivThumbnail)
                .load(item.path)
                .centerCrop()
                .into(binding.ivThumbnail)
            
            // 显示视频时长（如果是视频）
            if (item.duration > 0) {
                binding.tvDuration.visibility = android.view.View.VISIBLE
                binding.tvDuration.text = formatDuration(item.duration)
            } else {
                binding.tvDuration.visibility = android.view.View.GONE
            }
            
            // 显示收藏标记
            binding.ivFavorite.visibility = if (item.isFavorite) {
                android.view.View.VISIBLE
            } else {
                android.view.View.GONE
            }
            
            // 点击事件
            binding.root.setOnClickListener {
                onItemClick(item, adapterPosition)
            }
            
            // 长按事件（用于选择模式）
            binding.root.setOnLongClickListener {
                onItemLongClick(item)
            }
        }
        
        /**
         * 格式化视频时长
         * 例如：3665 -> "01:01:05" 或 "03:45"
         */
        private fun formatDuration(durationMs: Long): String {
            val seconds = (durationMs / 1000) % 60
            val minutes = (durationMs / (1000 * 60)) % 60
            val hours = durationMs / (1000 * 60 * 60)
            
            return if (hours > 0) {
                String.format("%02d:%02d:%02d", hours, minutes, seconds)
            } else {
                String.format("%02d:%02d", minutes, seconds)
            }
        }
    }
    
    class MediaDiffCallback : DiffUtil.ItemCallback<MediaItem>() {
        override fun areItemsTheSame(oldItem: MediaItem, newItem: MediaItem): Boolean {
            return oldItem.id == newItem.id
        }
        
        override fun areContentsTheSame(oldItem: MediaItem, newItem: MediaItem): Boolean {
            return oldItem == newItem
        }
    }
}
```

# 系统架构设计基础 - 架构风格

### 5.1 软件架构定义（IEEE 1471-2000）

> 架构是体现在组件中的系统的基本组织、组件彼此的关系与环境的关系及指导其设计和发展的原则。

**核心要素**：
- **组件**：系统核心功能单元
- **连接件**：组件间交互机制
- **约束**：设计规则
- **设计原则**：指导架构演化

---

### 5.2 五大架构风格

| 风格 | 核心特征 | 典型应用 |
|-----|---------|---------|
| **数据流风格** | 数据按流程传递，组件通过"输入-处理-输出"交互 | 日志分析、编译器 |
| **调用/返回风格** | 组件通过调用函数或方法交互，存在明确调用关系 | Web应用、管理系统 |
| **以数据为中心** | 组件围绕共享数据存储交互，数据是核心 | 数据库应用、缓存系统 |
| **虚拟机风格** | 通过虚拟机制模拟特定环境 | 解释器、Java虚拟机、Docker |
| **独立构件风格** | 组件独立运行，通过事件、消息松耦合交互 | 事件驱动、微服务 |

---

### 5.3 分层架构详解

**经典三层架构**：
```
┌─────────────────┐
│    表示层        │  用户界面、交互逻辑
├─────────────────┤
│   业务逻辑层      │  核心业务规则处理
├─────────────────┤
│   数据访问层      │  数据持久化、CRUD操作
├─────────────────┤
│     数据库        │  数据存储
└─────────────────┘
```

**分层原则**：
- 每层只依赖其下方层
- 上层调用下层，下层不能调用上层
- 通过接口解耦

---

### 5.4 微服务架构特点

**核心特征**：
- 服务拆分：按业务边界拆分
- 独立部署：每个服务独立部署升级
- 去中心化：去中心化管理数据
- 轻量通信：HTTP REST、gRPC

**优势**：
- 高并发：可独立扩容
- 可扩展：新增服务不影响其他服务
- 容错性：单服务故障不影响全局

**Q3: synchronized与ReentrantLock的区别？**

```java
// synchronized (JVM层面，自动释放锁)
synchronized (this) {
    // 临界区代码
}

// ReentrantLock (Java层面，手动控制)
private final ReentrantLock lock = new ReentrantLock();

public void method() {
    lock.lock();
    try {
        // 临界区代码
    } finally {
        lock.unlock();  // 必须手动释放
    }
}

// 主要区别：
// 1. synchronized自动释放，ReentrantLock需手动
// 2. ReentrantLock支持tryLock超时
// 3. ReentrantLock支持公平/非公平锁
// 4. ReentrantLock可绑定多个Condition
```

### 6.3 算法面试题

**Q4: 二叉树前序遍历（非递归）**

```java
public List<Integer> preorderTraversal(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;
    
    Stack<TreeNode> stack = new Stack<>();
    stack.push(root);
    
    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        result.add(node.val);  // 根
        
        // 先压右子节点，再压左子节点
        if (node.right != null) {
            stack.push(node.right);
        }
        if (node.left != null) {
            stack.push(node.left);
        }
    }
    
    return result;
}
```
