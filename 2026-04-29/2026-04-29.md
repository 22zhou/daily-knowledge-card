## 2.2 程序员搞副业的三个"反直觉"定律

### 定律1：先搞定流量，再写最后一行代码

技术人最大的通病是"自嗨"。正确姿势：
1. 哪怕只有一个想法，先花半天时间在CSDN或知乎写篇文章
2. 如果有人点赞评论问"老哥求个链接"，你再去动手写代码
3. 如果根本没人理，立刻放弃，切下一个点子

### 定律2：技术架构越土越好

做副业不是在公司搞技术评审，千万别为了炫技去上微服务、分布式、K8s全家桶。

**核心目的**：验证业务能不能赚钱！

哪怕是用单体SpringBoot、最古老的PHP、1核2G的轻量云服务器，只要能跑通核心业务流，那就是好架构。

### 定律3：不要追求完美，要追求闭环

代码永远写不完，Bug永远改不完。**完成比完美重要一万倍**。

第一版只要能跑通主流程，赶紧推上线挨骂。用户的真实反馈，才是你迭代的最佳驱动力。

## 4.2 C4模型：业界最推荐的架构图框架

C4模型由Simon Brown提出，是目前最受推崇的架构可视化框架，通过四个递进的层级来展示系统。

### 第一层：系统上下文图（Context）

**受众**：所有人，特别是非技术人员
**核心问题**：这个系统是做什么的？它和谁打交道？

```
┌─────────────────────────────────────┐
│          系统边界（黑盒）            │
│                                     │
│    ┌───────┐      ┌───────────┐    │
│    │ 用户A │      │ 外部系统B │    │
│    └───┬───┘      └─────┬─────┘    │
│        │                │          │
│        └────────┬────────┘          │
│                 │                  │
│            ┌────┴────┐              │
│            │ 你的系统 │              │
│            └─────────┘              │
└─────────────────────────────────────┘
```

### 第二层：容器图（Containers）

**受众**：开发、测试、运维人员
**核心问题**：系统的整体形态是什么？由哪些主要部分组成？

```
┌─────────────────────────────────────────────────────────┐
│                      你的系统                            │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐    │
│  │ Web应用  │  │ 移动App │  │ API服务  │  │ 数据库  │    │
│  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘    │
│       │            │            │            │         │
│       └────────────┴─────┬──────┴────────────┘         │
│                           │                             │
│                     ┌─────┴─────┐                       │
│                     │ API网关   │                       │
│                     └───────────┘                       │
└─────────────────────────────────────────────────────────┘
```

### 第三层：组件图（Components）

**受众**：开发人员、架构师
**核心问题**：这个服务内部是如何组织的？

```java
// Spring Boot 组件图示例
// Controller层：处理HTTP请求
@RestController
public class UserController {
    
    @Autowired
    private UserService userService;
    
    @PostMapping("/api/users")
    public ResponseEntity<UserDTO> createUser(@RequestBody UserRequest request) {
        // 验证请求
        ValidationUtils.validate(request);
        // 调用服务层
        UserDTO user = userService.createUser(request);
        return ResponseEntity.ok(user);
    }
}

// Service层：处理业务逻辑
@Service
public class UserService {
    
    @Autowired
    private UserRepository userRepository;
    
    @Transactional
    public UserDTO createUser(UserRequest request) {
        // 业务逻辑处理
        User user = new User();
        user.setUsername(request.getUsername());
        user.setEmail(request.getEmail());
        // 持久化
        return userRepository.save(user).toDTO();
    }
}

// Repository层：数据访问
@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByUsername(String username);
    Optional<User> findByEmail(String email);
}
```

### 核心流程图：图像编辑完整流程

```java
/**
 * Android 图像编辑完整流程代码示例
 * 从选择图片 → 编辑 → 保存的完整链路
 */

// 1. 图片选择器
public class ImagePicker {
    
    // 启动图片选择
    public void pickImage(Activity activity, int requestCode) {
        Intent intent = new Intent(Intent.ACTION_PICK);
        intent.setType("image/*");
        activity.startActivityForResult(intent, requestCode);
    }
    
    // 处理选择结果
    public void onActivityResult(int requestCode, int resultCode, Intent data) {
        if (resultCode == RESULT_OK && data != null) {
            Uri imageUri = data.getData();
            // 获取图片路径
            String imagePath = getPathFromUri(imageUri);
            // 传递给编辑模块
            launchImageEditor(imagePath);
        }
    }
}

// 2. 图像编辑核心类
public class ImageEditor {
    
    private Bitmap originalBitmap;
    private Bitmap currentBitmap;
    private Stack<EditAction> undoStack = new Stack<>();
    private Stack<EditAction> redoStack = new Stack<>();
    
    /**
     * 加载图片
     */
    public void loadImage(String imagePath) {
        originalBitmap = BitmapFactory.decodeFile(imagePath);
        currentBitmap = originalBitmap.copy(Bitmap.Config.ARGB_8888, true);
    }
    
    /**
     * 裁剪图片
     */
    public Bitmap crop(int left, int top, int width, int height) {
        // 保存操作到撤销栈
        undoStack.push(new EditAction("crop", currentBitmap));
        redoStack.clear();
        
        // 执行裁剪
        currentBitmap = Bitmap.createBitmap(
            currentBitmap, left, top, width, height
        );
        return currentBitmap;
    }
    
    /**
     * 旋转图片
     */
    public Bitmap rotate(float degrees) {
        Matrix matrix = new Matrix();
        matrix.postRotate(degrees);
        Bitmap rotated = Bitmap.createBitmap(
            currentBitmap, 0, 0, 
            currentBitmap.getWidth(), 
            currentBitmap.getHeight(),
            matrix, true
        );
        
        // 保存操作
        undoStack.push(new EditAction("rotate", currentBitmap));
        redoStack.clear();
        currentBitmap = rotated;
        
        return currentBitmap;
    }
    
    /**
     * 调整亮度
     */
    public Bitmap adjustBrightness(float factor) {
        // factor: 0.0 = 黑色, 1.0 = 原始, 2.0 = 白色
        ColorMatrix cm = new ColorMatrix();
        cm.setScale(factor, factor, factor, 1);
        
        Paint paint = new Paint();
        paint.setColorFilter(new ColorMatrixColorFilter(cm));
        
        Bitmap result = Bitmap.createBitmap(
            currentBitmap.getWidth(),
            currentBitmap.getHeight(),
            Bitmap.Config.ARGB_8888
        );
        
        Canvas canvas = new Canvas(result);
        canvas.drawBitmap(currentBitmap, 0, 0, paint);
        
        undoStack.push(new EditAction("brightness", currentBitmap));
        currentBitmap = result;
        
        return result;
    }
    
    /**
     * 调整饱和度
     */
    public Bitmap adjustSaturation(float saturation) {
        ColorMatrix cm = new ColorMatrix();
        cm.setSaturation(saturation);
        
        Paint paint = new Paint();
        paint.setColorFilter(new ColorMatrixColorFilter(cm));
        
        Bitmap result = Bitmap.createBitmap(
            currentBitmap.getWidth(),
            currentBitmap.getHeight(),
            Bitmap.Config.ARGB_8888
        );
        
        Canvas canvas = new Canvas(result);
        canvas.drawBitmap(currentBitmap, 0, 0, paint);
        
        undoStack.push(new EditAction("saturation", currentBitmap));
        currentBitmap = result;
        
        return result;
    }
    
    /**
     * 添加滤镜
     */
    public Bitmap applyFilter(FilterType filterType) {
        ColorMatrix colorMatrix = null;
        
        switch (filterType) {
            case GRAYSCALE:
                colorMatrix = new ColorMatrix();
                colorMatrix.setSaturation(0);
                break;
            case SEPIA:
                colorMatrix = new ColorMatrix();
                colorMatrix.set(new float[] {
                    0.393f, 0.769f, 0.189f, 0, 0,
                    0.349f, 0.686f, 0.168f, 0, 0,
                    0.272f, 0.534f, 0.131f, 0, 0,
                    0, 0, 0, 1, 0
                });
                break;
            case WARM:
                colorMatrix = new ColorMatrix();
                colorMatrix.set(new float[] {
                    1.2f, 0, 0, 0, 20,
                    0, 1.0f, 0, 0, 10,
                    0, 0, 0.8f, 0, -20,
                    0, 0, 0, 1, 0
                });
                break;
        }
        
        return adjustWithColorMatrix(colorMatrix);
    }
    
    /**
     * 撤销
     */
    public void undo() {
        if (!undoStack.isEmpty()) {
            EditAction action = undoStack.pop();
            redoStack.push(new EditAction("redo", currentBitmap));
            currentBitmap = action.savedBitmap;
        }
    }
    
    /**
     * 重做
     */
    public void redo() {
        if (!redoStack.isEmpty()) {
            EditAction action = redoStack.pop();
            undoStack.push(new EditAction("undo", currentBitmap));
            currentBitmap = action.savedBitmap;
        }
    }
    
    /**
     * 保存编辑结果
     */
    public String saveImage(String outputPath, int quality) {
        try {
            FileOutputStream fos = new FileOutputStream(outputPath);
            currentBitmap.compress(Bitmap.CompressFormat.JPEG, quality, fos);
            fos.close();
            return outputPath;
        } catch (IOException e) {
            e.printStackTrace();
            return null;
        }
    }
}

/**
 * 滤镜类型枚举
 */
public enum FilterType {

    GRAYSCALE,
    SEPIA,
    WARM,
    COOL,
    VINTAGE,
    DRAMATIC
}

 * 编辑操作记录（用于撤销/重做）
 */
public class EditAction {
    public String actionType;
    public Bitmap savedBitmap;
    
    public EditAction(String type, Bitmap bitmap) {
        this.actionType = type;
        this.savedBitmap = bitmap.copy(Bitmap.Config.ARGB_8888, true);
    }
```

## 系统架构设计师 · 软件架构风格

### 5.1 什么是软件架构风格？

软件架构风格是**对系统组件结构的高层抽象**，定义了组件之间的通信模式和协作方式。

**核心作用**：
- 提供**可重用**的设计方案
- 促进**设计复用**和**系统演化**
- 定义**组件交互**的规则和约束

---

### 5.2 主要架构风格分类

#### 一、经典分层架构（Layered Architecture）

```
┌────────────────────────────────┐
│       表现层 (Presentation)    │
│    处理用户界面和用户请求       │
└──────────────┬───────────────┘
               │调用
┌──────────────▼───────────────┐
│       业务层 (Business)       │
│    实现核心业务逻辑            │
└──────────────┬───────────────┘
               │调用
┌──────────────▼───────────────┐
│       数据层 (Data Access)   │
│    处理数据持久化和访问        │
└──────────────┬───────────────┘
               │访问
┌──────────────▼───────────────┐
│           数据库              │
└─────────────────────────────┘
```

**特点**：
- 每层只与**相邻层**交互
- 职责清晰，易于维护
- 依赖方向单一（上层依赖下层）

**适用场景**：企业应用、桌面应用、移动应用

#### 二、客户端-服务器架构（C/S）

```
┌──────────┐                    ┌──────────┐
│  客户端1  │                    │  客户端2  │
└─────┬────┘                    └─────┬────┘
      │                              │
      │     网络通信 (TCP/HTTP)       │
      └──────────────┬───────────────┘
                     │
              ┌──────▼──────┐
              │    服务器    │
              │ (处理业务)   │
              └──────┬──────┘
                     │
              ┌──────▼──────┐
              │    数据库    │
              └─────────────┘
```

**变体**：
- **两层C/S**：客户端+服务器+数据库
- **三层C/S**：表现层+业务层+数据层

**特点**：
- 集中管理数据
- 客户端承担部分处理
- 网络依赖性强

#### 三、微服务架构（Microservices）

```
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│用户服务  │  │订单服务  │  │支付服务  │  │物流服务  │
└────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘
     │            │            │            │
     └────────────┴─────┬───────┴────────────┘
                       │
              ┌────────▼────────┐
              │   API网关       │
              └────────┬────────┘
                       │
     ┌─────────────────┼─────────────────┐
     │                 │                 │
┌────▼────┐      ┌─────▼────┐     ┌─────▼────┐
│  MySQL  │      │  Redis   │     │   Kafka  │
│ (用户DB)│      │ (缓存)   │     │ (消息队列)│
└─────────┘      └──────────┘     └──────────┘
```

**特点**：
- **服务独立部署**
- **去中心化管理**
- **故障隔离**
- **技术异构**

**适用场景**：大型互联网应用、云原生应用

#### 四、事件驱动架构（EDA）

```
┌──────────┐    事件发布     ┌──────────┐
│  事件源   │ ─────────────> │  事件总线  │
└──────────┘                └────┬─────┘
                                  │
           ┌──────────────────────┼──────────────────────┐
           │                      │                      │
     ┌─────▼─────┐          ┌─────▼─────┐          ┌─────▼─────┐
     │ 处理器1   │          │ 处理器2   │          │ 处理器3   │
     └───────────┘          └───────────┘          └───────────┘
```

**常见模式**：
- **发布-订阅**（Pub/Sub）
- **事件溯源**（Event Sourcing）
- **CQRS**（命令查询职责分离）

**适用场景**：实时系统、物联网、日志处理

#### 五、管道-过滤器架构

```
┌───────┐   数据流   ┌────────┐   数据流   ┌────────┐   数据流   ┌───────┐
│ 输入源 │ ────────> │过滤器1 │ ────────> │过滤器2 │ ────────> │ 输出  │
└───────┘            └────────┘            └────────┘            └───────┘
                            │                                     │
                     ┌──────▼──────┐                              │
                     │   过滤器3   │                              │
                     └─────────────┘
```

**特点**：
- 每个过滤器**独立工作**
- 数据**线性流动**
- 易于**扩展**和**复用**

**适用场景**：数据处理管道、编译器、图像处理

#### 六、领域驱动设计（DDD）架构

```
┌─────────────────────────────────────────────────────────┐
│                    限界上下文 (Bounded Context)           │
│  ┌─────────────────┐  ┌─────────────────┐               │
│  │    聚合根       │  │    聚合根       │               │
│  │  (Order)        │  │  (Customer)     │               │
│  │  ┌───────────┐  │  │  ┌───────────┐  │               │
│  │  │ 实体/值对象│  │  │  │ 实体/值对象│  │               │
│  │  └───────────┘  │  │  └───────────┘  │               │
│  │  ┌───────────┐  │  │  └───────────┘  │               │
│  │  │   领域服务 │  │  │                 │               │
│  │  └───────────┘  │  │                 │               │
│  └────────┬────────┘  └─────────────────┘               │
│           │                                                  │
│  ┌────────▼────────┐                                       │
│  │  仓储接口 (Repository) │                                │
│  └────────┬────────┘                                       │
└───────────┼─────────────────────────────────────────────────┘
            │
┌───────────▼─────────────────────────────────────────────────┐
│                     应用层 (Application)                      │
│  ┌─────────────────┐  ┌─────────────────┐                   │
│  │    应用服务      │  │     命令处理器  │                   │
│  └─────────────────┘  └─────────────────┘                   │
└──────────────────────────────────────────────────────────────┘
```

**核心概念**：
- **聚合**（Aggregate）
- **实体**（Entity）
- **值对象**（Value Object）
- **领域事件**（Domain Event）
- **限界上下文**（Bounded Context）

### 5.3 架构风格对比

| 风格 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 分层架构 | 清晰、易维护 | 层级多、影响性能 | 企业应用 |
| C/S架构 | 交互性好 | 客户端维护成本高 | 桌面应用 |

| DDD | 业务聚焦、可维护 | 学习曲线陡峭 | 复杂业务系统 |

### 题目：如何判断一棵二叉树是否是平衡二叉树？

```java
/**
 * 平衡二叉树定义：左右子树高度差不超过1，且左右子树都是平衡二叉树
 * 
 * 解题思路：
 * 1. 后序遍历，从底向上计算高度
 * 2. 一边计算一边判断是否平衡
 * 3. 时间复杂度：O(n)，空间复杂度：O(h)
 */
public class BalanceTree {
    
    // 定义二叉树节点
    static class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;
        
        TreeNode(int val) {
            this.val = val;
        }
    }
    
    /**
     * 方法一：暴力递归（自顶向下）
     * 时间复杂度：O(n²) - 每个节点都要遍历其所有子树
     */
    public boolean isBalanced_v1(TreeNode root) {
        if (root == null) return true;
        
        // 计算左右子树高度
        int leftHeight = getHeight(root.left);
        int rightHeight = getHeight(root.right);
        
        // 检查当前节点是否平衡
        if (Math.abs(leftHeight - rightHeight) > 1) {
            return false;
        }
        
        // 递归检查左右子树
        return isBalanced_v1(root.left) && isBalanced_v1(root.right);
    }
    
    private int getHeight(TreeNode node) {
        if (node == null) return 0;
        return Math.max(getHeight(node.left), getHeight(node.right)) + 1;
    }
    
    /**
     * 方法二：优化递归（自底向上）
     * 时间复杂度：O(n) - 每个节点只访问一次
     */
    public boolean isBalanced_v2(TreeNode root) {
        return height(root) != -1;
    }
    
    private int height(TreeNode node) {
        if (node == null) return 0;
        
        // 递归计算左右子树高度
        int leftHeight = height(node.left);
        int rightHeight = height(node.right);
        
        // 如果左子树不平衡，直接返回-1
        if (leftHeight == -1) return -1;
        
        // 如果右子树不平衡，直接返回-1
        if (rightHeight == -1) return -1;
        
        // 如果当前节点不平衡，返回-1
        if (Math.abs(leftHeight - rightHeight) > 1) {
            return -1;
        }
        
        // 返回当前节点的高度
        return Math.max(leftHeight, rightHeight) + 1;
    }
    
    // 测试
    public static void main(String[] args) {
        BalanceTree solution = new BalanceTree();
        
        // 构造平衡二叉树
        TreeNode root = new TreeNode(1);
        root.left = new TreeNode(2);
        root.right = new TreeNode(3);
        root.left.left = new TreeNode(4);
        root.left.right = new TreeNode(5);
        
        System.out.println("平衡二叉树: " + solution.isBalanced_v2(root)); // true
        
        // 构造非平衡二叉树
        TreeNode root2 = new TreeNode(1);
        root2.left = new TreeNode(2);
        root2.left.left = new TreeNode(3);
        root2.left.left.left = new TreeNode(4);
        
        System.out.println("非平衡二叉树: " + solution.isBalanced_v2(root2)); // false
    }
}
```

### 题目：LRU缓存实现

```java
import java.util.*;

/**
 * LRU (Least Recently Used) 缓存
 * 
 * 核心思想：
 * 1. 使用HashMap存储键值对，O(1)查找
 * 2. 使用双向链表维护访问顺序，最近访问的放在头部
 * 3. 当容量满时，删除链表尾部（最久未使用）
 * 
 * API：
 * - get(key): 获取值，如果存在则移动到头部
 * - put(key, value): 插入或更新，如果存在则更新值并移动到头部
 */
public class LRUCache {
    
    // 双向链表节点
    class Node {
        int key;
        int value;
        Node prev;
        Node next;
        
        Node(int key, int value) {
            this.key = key;
            this.value = value;
        }
    }
    
    private int capacity;  // 缓存容量
    private Map<Integer, Node> cache;  // HashMap存储
    private Node head;  // 虚拟头节点
    private Node tail;  // 虚拟尾节点
    
    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.cache = new HashMap<>();
        
        // 初始化虚拟头尾节点
        head = new Node(0, 0);
        tail = new Node(0, 0);
        head.next = tail;
        tail.prev = head;
    }
    
    /**
     * 获取缓存值
     * 如果存在，将节点移动到链表头部
     */
    public int get(int key) {
        Node node = cache.get(key);
        if (node == null) {
            return -1;  // 不存在
        }
        
        // 将节点移动到头部（最近使用）
        moveToHead(node);
        return node.value;
    }
    
    /**
     * 插入或更新缓存
     * 如果已存在，更新值并移动到头部
     * 如果不存在，新插入到头部，容量满则删除尾部
     */
    public void put(int key, int value) {
        Node node = cache.get(key);
        
        if (node != null) {
            // 已存在：更新值，移动到头部
            node.value = value;
            moveToHead(node);
        } else {
            // 不存在：新插入
            Node newNode = new Node(key, value);
            cache.put(key, newNode);
            addToHead(newNode);
            
            // 如果容量超限，删除最久未使用的
            if (cache.size() > capacity) {
                Node removed = removeTail();
                cache.remove(removed.key);
            }
        }
    }
    
    /**
     * 添加节点到链表头部
     */
    private void addToHead(Node node) {
        node.prev = head;
        node.next = head.next;
        head.next.prev = node;
        head.next = node;
    }
    
    /**
     * 移除节点
     */
    private void removeNode(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }
    
    /**
     * 移动节点到头部
     */
    private void moveToHead(Node node) {
        removeNode(node);
        addToHead(node);
    }
    
    /**
     * 删除链表尾部节点（最久未使用）
     */
    private Node removeTail() {
        Node node = tail.prev;
        removeNode(node);
        return node;
    }
    
    // 测试
    public static void main(String[] args) {
        LRUCache cache = new LRUCache(2);
        
        cache.put(1, 1);    // 缓存: {1=1}
        cache.put(2, 2);    // 缓存: {1=1, 2=2}
        System.out.println(cache.get(1)); // 返回1，缓存: {2=2, 1=1}
        
        cache.put(3, 3);    // 容量满，删除2，缓存: {1=1, 3=3}
        System.out.println(cache.get(2)); // 返回-1（不存在）
        
        cache.put(4, 4);    // 删除1，缓存: {3=3, 4=4}
        System.out.println(cache.get(1)); // 返回-1（不存在）
        System.out.println(cache.get(3));  // 返回3
        System.out.println(cache.get(4)); // 返回4
    }
}

/**
```
```java
 * 1. MeasureSpec - 测量规格
 * 
 * 三种模式：
 * - UNSPECIFIED (0)：父容器不限制，子视图想要多大就给多大
 * - EXACTLY (1)：父容器精确指定大小，子视图必须服从
 * - AT_MOST (2)：子视图最大不能超过指定值
 */
public class MeasureSpecDemo {
    
    // MeasureSpec的创建
    public static int makeMeasureSpec(int size, int mode) {
        return size + mode;
    }
    
    // 获取模式
    public static int getMode(int measureSpec) {
        return measureSpec & MODE_MASK;
    }
    
    // 获取尺寸
    public static int getSize(int measureSpec) {
        return measureSpec & ~MODE_MASK;
    }
    
    // 使用示例
    @Override
    protected void onMeasure(int widthMeasureSpec, int heightMeasureSpec) {
        int widthMode = MeasureSpec.getMode(widthMeasureSpec);
        int widthSize = MeasureSpec.getSize(widthMeasureSpec);
        
        int heightMode = MeasureSpec.getMode(heightMeasureSpec);
        int heightSize = MeasureSpec.getSize(heightMeasureSpec);
        
        int width = 0;
        int height = 0;
        
        // 根据不同模式计算尺寸
        if (widthMode == MeasureSpec.EXACTLY) {
            width = widthSize;  // 使用精确值（match_parent或具体dp）
        } else if (widthMode == MeasureSpec.AT_MOST) {
            width = Math.min(widthSize, getDesiredWidth());  // wrap_content
        }
        
        // 记住调用setMeasuredDimension
        setMeasuredDimension(width, height);
    }

public class CustomCircleView extends View {

    private int color = Color.BLUE;
    private int radius = 100;

        super(context);
        init();
    }

        paint = new Paint();
        paint.setAntiAlias(true);
        paint.setColor(color);
        paint.setStyle(Paint.Style.FILL);
    }
    
    // 1. 测量阶段
    @Override
    protected void onMeasure(int widthMeasureSpec, int heightMeasureSpec) {
        super.onMeasure(widthMeasureSpec, heightMeasureSpec);
        
        int widthMode = MeasureSpec.getMode(widthMeasureSpec);
        int widthSize = MeasureSpec.getSize(widthMeasureSpec);
        
        int heightMode = MeasureSpec.getMode(heightMeasureSpec);
        int heightSize = MeasureSpec.getSize(heightMeasureSpec);
        
        // 如果是wrap_content，使用默认半径的2倍作为尺寸
        int desiredWidth = radius * 2 + getPaddingLeft() + getPaddingRight();
        int desiredHeight = radius * 2 + getPaddingTop() + getPaddingBottom();
        
        int width = resolveSize(desiredWidth, widthMeasureSpec);
        int height = resolveSize(desiredHeight, heightMeasureSpec);
        
        setMeasuredDimension(width, height);
    }

    protected void onLayout(boolean changed, int left, int top, int right, int bottom) {

    }

    @Override
    protected void onDraw(Canvas canvas) {
        super.onDraw(canvas);
        
        int centerX = getWidth() / 2;
        int centerY = getHeight() / 2;
        
        // 绘制圆形
        canvas.drawCircle(centerX, centerY, radius, paint);
```
