### Android相册图像编辑开发：手势缩放与拖拽实现

在相册应用中，图像的缩放和拖拽是最基础也最核心的功能。本文将详细介绍如何实现一个流畅的图片查看器。

#### 一、核心原理

1. **Matrix变换**：Android中使用Matrix来实现图片的缩放、旋转、平移
2. **手势检测**：通过ScaleGestureDetector检测缩放手势，通过GestureDetector检测滑动
3. **边界控制**：防止图片缩到太小或拖出屏幕边界

#### 二、完整代码实现

```java
public class ZoomableImageView extends AppCompatImageView {

    private Matrix matrix = new Matrix();
    private Matrix savedMatrix = new Matrix();
    
    // 模式：NONE-无操作, DRAG-拖拽, ZOOM-缩放
    private static final int NONE = 0;
    private static final int DRAG = 1;
    private static final int ZOOM = 2;
    private int mode = NONE;
    
    private PointF start = new PointF();
    private PointF mid = new PointF();
    private float oldDist = 1f;
    
    private float minScale = 1f;
    private float maxScale = 5f;
    
    private ScaleGestureDetector scaleGestureDetector;
    private GestureDetector gestureDetector;

    public ZoomableImageView(Context context) {
        super(context);
        init(context);
    }

    private void init(Context context) {
        setScaleType(ScaleType.MATRIX);
        scaleGestureDetector = new ScaleGestureDetector(context, new ScaleListener());
        gestureDetector = new GestureDetector(context, new GestureListener());
    }

    @Override
    public boolean onTouchEvent(MotionEvent event) {
        scaleGestureDetector.onTouchEvent(event);
        gestureDetector.onTouchEvent(event);

        switch (event.getAction() & MotionEvent.ACTION_MASK) {
            case MotionEvent.ACTION_DOWN:
                savedMatrix.set(matrix);
                start.set(event.getX(), event.getY());
                mode = DRAG;
                break;

            case MotionEvent.ACTION_POINTER_DOWN:
                oldDist = spacing(event);
                if (oldDist > 10f) {
                    savedMatrix.set(matrix);
                    midPoint(mid, event);
                    mode = ZOOM;
                }
                break;

            case MotionEvent.ACTION_UP:
            case MotionEvent.ACTION_POINTER_UP:
                mode = NONE;
                break;

            case MotionEvent.ACTION_MOVE:
                if (mode == DRAG) {
                    matrix.set(savedMatrix);
                    matrix.postTranslate(event.getX() - start.x, event.getY() - start.y);
                } else if (mode == ZOOM && event.getPointerCount() >= 2) {
                    float newDist = spacing(event);
                    if (newDist > 10f) {
                        matrix.set(savedMatrix);
                        float scale = newDist / oldDist;
                        matrix.postScale(scale, scale, mid.x, mid.y);
                    }
                }
                break;
        }
        fixTranslation();
        setImageMatrix(matrix);
        return true;
    }

    private float spacing(MotionEvent event) {
        if (event.getPointerCount() < 2) return 0;
        float x = event.getX(0) - event.getX(1);
        float y = event.getY(0) - event.getY(1);
        return (float) Math.sqrt(x * x + y * y);
    }

    private void midPoint(PointF point, MotionEvent event) {
        if (event.getPointerCount() < 2) return;
        point.set((event.getX(0) + event.getX(1)) / 2,
                  (event.getY(0) + event.getY(1)) / 2);
    }

    private void fixTranslation() {
        float[] values = new float[9];
        matrix.getValues(values);
        float scale = values[Matrix.MSCALE_X];
        if (scale < minScale) {
            matrix.postScale(minScale / scale, minScale / scale, getWidth() / 2f, getHeight() / 2f);
        } else if (scale > maxScale) {
            matrix.postScale(maxScale / scale, maxScale / scale, getWidth() / 2f, getHeight() / 2f);
        }
    }

    private class ScaleListener extends ScaleGestureDetector.SimpleOnScaleGestureListener {}

    private class GestureListener extends GestureDetector.SimpleOnGestureListener {
        @Override
        public boolean onDoubleTap(MotionEvent e) {
            float[] values = new float[9];
            matrix.getValues(values);
            float curScale = values[Matrix.MSCALE_X];
            if (curScale > minScale * 1.5f) {
                matrix.reset();
            } else {
                matrix.postScale(2.5f / curScale, 2.5f / curScale, e.getX(), e.getY());
            }
            setImageMatrix(matrix);
            return true;
        }
    }
}
```

| 功能 | 实现方式 |
|------|----------|

## 📚 软考备考

### 系统架构设计师 - 可靠性设计

#### 一、可靠性基本概念

**可靠性**：系统在规定时间内、规定条件下完成规定功能的能力

**可用性**：系统在任意时刻可使用的概率

```
可用性 = MTBF / (MTBF + MTTR)
```

- MTBF：平均无故障时间
- MTTR：平均修复时间

**可维护性**：系统发生故障后能恢复正常工作的难易程度

#### 二、冗余技术

**1. 主动冗余**
- 通过多台设备同时运行，结果取多数或平均值
- 特点：故障时自动切换，无需等待

**2. 被动冗余**
- 主设备故障后启动备份设备
- 热备份：备份设备实时同步，立即切换
- 冷备份：备份设备定期同步，切换需要时间

**3. 混合冗余**
- 组合主动和被动冗余
- 正常时主动冗余工作，故障时切换到备份

#### 三、软件容错

| 技术 | 说明 |
|------|------|
| N版本程序 | 多版本独立开发，投票输出 |
| 恢复块 | 主版本+多个备份版本，自动恢复 |
| 检查点/回滚 | 定期保存状态，故障时回滚 |

#### 四、集群技术

**1. 高可用集群 (HA)**
- 双机热备：一主一备，故障自动切换
- 多机集群：3台以上，互为备份

**2. 负载均衡集群**
- DNS轮询：简单但无法健康检查
- LVS：Linux虚拟服务器，4层/7层负载均衡
- Nginx：反向代理，7层负载均衡

**3. 高性能集群**
- 并行计算：MPI、MapReduce
- 分布式存储：HDFS、Ceph

#### 五、案例分析要点

在下午考试中，可靠性设计题目通常要求：
1. 分析系统可靠性需求
2. 选择合适的冗余策略
3. 计算系统可用性指标
4. 提出改进建议

### 一、数据结构

**Q1: HashMap的扩容机制是怎样的？**

HashMap使用"数组+链表+红黑树"结构：
- 默认容量16，负载因子0.75
- 元素数量超过 容量×0.75 时触发扩容
- 扩容为原来的2倍
- 重新计算所有元素的hash位置（重新hash）
- 链表长度超过8且数组长度>=64时，链表转为红黑树

**Q2: 为什么HashMap的容量是2的幂次？**

核心原因：位运算效率高于取模运算

```java
// 源码中的优化
// index = (n - 1) & hash
// 当 n = 2^k 时，(n-1) 的二进制是连续的1
// 与hash进行&运算等价于 hash % n，但速度更快
```

### 二、算法

**Q3: 如何在O(n)时间内找到数组中第K大的元素？**

快速选择算法（QuickSelect）：

```python
def find_kth_largest(nums, k):
    import random
    
    def partition(left, right, pivot_index):
        pivot = nums[pivot_index]
        nums[pivot_index], nums[right] = nums[right], nums[pivot_index]
        store_index = left
        
        for i in range(left, right):
            if nums[i] > pivot:  # 降序排列
                nums[store_index], nums[i] = nums[i], nums[store_index]
                store_index += 1
        
        nums[right], nums[store_index] = nums[store_index], nums[right]
        return store_index
    
    left, right = 0, len(nums) - 1
    target = k - 1  # 第K大 = 第K-1小
    
    while True:
        pivot_index = random.randint(left, right)
        pivot_index = partition(left, right, pivot_index)
        
        if pivot_index == target:
            return nums[pivot_index]
        elif pivot_index < target:
            left = pivot_index + 1
        else:
            right = pivot_index - 1
```

### 三、系统设计

**Q4: 设计一个短链接系统**

```
┌─────────────────────────────────────────────────┐
│                     短链接系统                    │
├─────────────────────────────────────────────────┤
│                                                 │
│   用户请求: https://t.cn/abc123                   │
│                    ↓                            │
│         ┌───────────────────┐                  │
│         │   重定向服务        │                  │
│         │   302/301跳转      │                  │
│         └───────────────────┘                  │
│                    ↓                            │
│         ┌───────────────────┐                  │
│         │   缓存层 (Redis)   │                  │
│         │   Key: abc123     │                  │
│         └───────────────────┘                  │
│                    ↓ 未命中                     │
│         ┌───────────────────┐                  │
│         │   数据库 (MySQL)   │                  │
│         │   长链接存储        │                  │
│         └───────────────────┘                  │
│                                                 │
└─────────────────────────────────────────────────┘
```

**核心代码**：

```python
import hashlib

class ShortUrlService:
    def __init__(self, db, cache):
        self.db = db
        self.cache = cache
        self.base_url = "https://t.cn/"
    
    def to_short_url(self, long_url):
        # 1. 计算hash
        hash_str = hashlib.md5(long_url.encode()).hexdigest()
        # 2. 取前6位转为62进制
        key = base62_encode(int(hash_str[:8], 16))[:6]
        # 3. 存储映射关系
        self.db.set(key, long_url)
        self.cache.set(key, long_url, ttl=3600*24)
        return self.base_url + key
    
    def to_long_url(self, short_url):
        key = short_url.replace(self.base_url, "")
        # 1. 先查缓存
        long_url = self.cache.get(key)
        if long_url:
            return long_url
        # 2. 缓存未命中，查数据库
        long_url = self.db.get(key)
        if long_url:
            self.cache.set(key, long_url)
        return long_url
```

### 四、Android

**Q5: Handler机制是怎样的？**

```
┌─────────────────────────────────────────────────┐
│                  Handler 机制                    │
├─────────────────────────────────────────────────┤
│                                                 │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐      │
│   │ Message │    │ Message │    │ Message │      │
│   │  Queue  │←───│ Queue   │←───│  (obj) │      │
│   └────┬────┘    └────┬────┘    └─────────┘      │
│        │               │                        │
│        ↓               │                        │
│   ┌─────────┐    ┌─────┴────┐                    │
│   │ Looper  │←───│  Message │                    │
│   │ (线程1) │    │  Queue   │                    │
│   └────┬────┘    └──────────┘                    │
│        │                                       │
│        ↓                                       │
│   ┌─────────┐                                  │
│   │Handler A│  处理消息                          │
│   └─────────┘                                  │
│                                                 │
└─────────────────────────────────────────────────┘
```

**关键点**：
1. 每个线程只有一个Looper
2. Looper.loop()不断从MessageQueue取消息
3. Handler发送消息到MessageQueue
4. Handler处理消息（handleMessage）
